## Suggested Event Storming Use

1. Start from an action type that represents a user intention, for example `UI/COMMENT/CREATE`.
2. Read the producer column to find the screen/view model entry point.
3. Read the listener column to find orchestration.
4. Read the reducer column to find read-model state transitions.
5. Treat actions with many listeners or reducers as candidates for explicit process naming.
